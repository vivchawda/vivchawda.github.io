Background

Project Overview

RADAR is a legacy tool used by service technicians for over 10 years. The redesign aimed to optimize workflows, reduce resolution time, and lower operational costs while maintaining user confidence during the transition.

Objectives

  • Increase number of issues resolved by lowering resolution time for each issue
  • Lower overhead costs for company in resolving issues and customer complaints
  • Gain confidence among users to move away from legacy workflow and adopt optimized workflows
  • Design a new workflow to optimize time taken to shortlist, analyse and address an issue

Challenges

  • Users are accustomed to using the legacy tool unchanged for 10+ years and are reluctant to change
  • Complex workflows across multiple tools (RADAR, CAT, ClickScheduler)
  • High frequency of field visits adding to operational costs
  • Need to optimize workflow without disrupting established mental models

Key Research Findings

After extensive interviews across three modalities and global regions, findings were classified into three design principle categories:

1

Structured Information

Information should be easy to locate with primary information upfront and higher priority. Critical information should be immediately visible while secondary information can be discreet.

  • "Too many countries shown here. I am only concerned with Spain."
  • "Before going into details, I need to check system reachability first"
2

Adaptable Interface

Interface should allow customizations so users can tailor according to their style. Enable different workflows by different users and understand user based on login credentials.

  • "I cannot customize the columns in the table"
  • "Everything resets on re-login"
3

Optimized Navigation

Navigation structure should be intuitive and clear, correlating with typical user workflow. Use visual aids and keyboard shortcuts to help users with quicker task completion.

  • "I get lost with tabs. Where am I?"
  • "Need to navigate several pages to carry on with workflow"

Methodology

Design Process

Followed an Agile methodology with iterative design cycles, moving from research insights through to tested and validated solutions.

Phase 1

Background & Research

Conducted stakeholder interviews, comprehensive user research across global regions, legacy tool audit, and defined scope & objectives based on business and user needs.

Phase 2

Definition

Established design principles (Structured Information, Adaptable Interface, Optimized Navigation), created information architecture for different modalities (CV, MR, CT), and mapped interaction flows.

Phase 3

Design Exploration

Created wireframe sketches exploring different layout approaches, developed visual design system with consistent patterns, and built interactive prototypes for user validation.

Phase 4

Test & Iterations

Conducted user testing sessions with remote service engineers and modality managers, iterated designs based on feedback, and created detailed design specifications for development.

Phase 5

Launch & Next Steps

Developed incremental rollout plan starting with single user MVP, provided ongoing development support, and planned feature additions across modalities with predictive engine integration.

Optimized Navigation Models

Designed two primary navigation workflows based on user research:

Workflow 1: Region-Based Search

Login ‚Üí Dashboard ‚Üí Details ‚Üí Analyse LOG ‚Üí Create Case ‚Üí Close ‚Üí Next

User searches their allocated region for systems with issues, views system health and reachability, downloads LOG files, creates cases with pre-filled data, and moves to next system with removed dealt systems from list.

Workflow 2: Direct SRN Search

Login ‚Üí Dashboard ‚Üí SEARCH ‚Üí Details ‚Üí Analyse LOG ‚Üí Create Case ‚Üí Close ‚Üí Next

User directly searches by System Reference Number (SRN), views system status and error details, analyses LOG file with visual timeline, creates/opens cases using templates, and efficiently moves through systems.

Final Design

Solution & Key Features

The redesigned RADAR interface prioritizes critical information, streamlines workflows, and provides powerful analysis tools while maintaining familiarity for existing users.

üéØ Dashboard Overview

Location-based system navigation with global statistics showing systems processed, critical systems, and resolution metrics. Monitoring dashboard displays prioritized alerts with aggregate titles, severity indicators, and last alert dates.

üìä Visual LOG Timeline

Revolutionary visual timeline representation of LOG files that charts activities and alerts chronologically. Interactive cursor allows users to jump directly to error lines, drastically reducing analysis time.

üìÖ System Health Calendar

Monthly calendar visualization showing daily system health status using three-state colors and icons. Technicians can immediately identify problematic days, field service engineer visits, and usage trends at a glance.

üîç Enhanced System Details

Comprehensive system information page with contract details, model specifications, open alerts, and system usage patterns. Filtered defects view with priority indicators. Notes section for team collaboration.

üìà Global Statistics Panel

Side panel targeted at Modality Managers showing overall modality performance metrics including total systems processed, number of critical systems, and successfully analyzed systems.

‚ö° Smart Case Creation

Streamlined case creation with modal dialog boxes that intelligently pre-fill critical information from system context. Integration with ClickScheduler for field service engineer scheduling.

Key Learnings & Takeaways

üéì Change Management

Successfully transitioning users from a 10+ year legacy tool requires incremental rollout, extensive user involvement, and maintaining familiarity while introducing improvements.

üåç Global User Research

Conducting research across multiple regions (Americas, Europe, APAC) and modalities (IxR, MR, CT) revealed both universal pain points and regional variations in workflows.

üìä Visual Information Design

The visual LOG timeline and system health calendar proved that thoughtful data visualization can dramatically reduce analysis time and cognitive load.

üîÑ Iterative Validation

Having in-house users allowed for rapid iteration and validation. Minimal intervention observation confirmed users could successfully navigate new paradigms.

Business Impact

The redesigned RADAR tool successfully met business objectives by lowering resolution time per issue, reducing overhead costs through decreased field visit frequency, and gaining user confidence to adopt the new optimized workflows. The phased rollout approach minimized disruption while the legacy tool was gradually deprecated, ensuring business continuity throughout the transition.
